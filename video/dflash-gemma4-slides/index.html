03 / MECHANISM 速くなる理由

先読みが当たるほど速い。外れるほど遅い。

DRAFT
軽い draft が次の token 群をまとめて提案する。
→
VERIFY
target が提案 token を検証し、合う部分だけ採用する。
→
ACCEPT
accepted が高いほど、1回の target 実行で進める token が増える。
今回の DFlash は block size 16。平均 accepted が 6 前後なら速い。3 前後まで落ちると、draft と rollback のコストで勝ちにくい。

07 / WHY NOT 6X? 公式値との差分

公式の “up to 6x” は、今回の条件そのものではない。

6x

公式値は “up to”。主に DFlash が得意なモデル・タスク・backend・hardware の組み合わせで出る上限として読むべき。

差分今回
backendMLX simple implementation
