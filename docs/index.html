Gemma 4 MTP Benchmark

A small reproducible benchmark for comparing Gemma 4 LiteRT-LM generation with Multi-Token Prediction enabled and disabled.

Speed Difference Video

A 30-second HyperFrames render shows the E4B coding sprint and the wider task matrix, making the MTP on/off difference easier to feel before running it locally.

Quick Start

git clone https://github.com/Sunwood-ai-labs/gemma4-mtp-benchmark.git
cd gemma4-mtp-benchmark
uv venv
uv pip install -e '.[bench]'
uv run gemma4-mtp-bench run --model e2b --backend gpu --prompt-set quick

Interactive Chat

Type your own prompts with LiteRT-LM MTP enabled, or compare MTP off and on.

uv run gemma4-mtp-bench chat --model e4b --backend gpu --mode mtp
uv run gemma4-mtp-bench chat --model e4b --backend gpu --mode compare

Local M1 Max Snapshot

In a one-round local run, predictable tasks showed the clearest gains: E4B GPU coding reached about 1.87x, JSON formatting about 1.65x, and log extraction about 1.51x. Creative and very short prompts were flat or slower.

DFlash Comparison

The repo now includes a separate MLX lane for z-lab/dflash. On the same M1 Max, Qwen3.5-4B with its DFlash draft reached 44.28 tok/s versus 28.31 tok/s baseline on an 8-sample gsm8k smoke test. The heavier Gemma 4 31B smoke ran too, but was flat on this machine: 11.38 tok/s DFlash versus 12.10 tok/s baseline. A longer mt-bench run at 512 max tokens did not reverse that: Gemma 4 31B measured 4.87 tok/s DFlash versus 8.87 tok/s baseline, while Qwen3.5-4B was nearly flat. 

scripts/run-dflash-mlx.sh

DFlash Contrast

A focused Qwen3.5-4B prompt experiment makes the mechanism visible: predictable math reached 1.86x with mean accepted length 6.59, structured JSON reached 1.44x with 5.14 accepted, and open-ended creative writing slowed to 0.93x with 3.62 accepted.
