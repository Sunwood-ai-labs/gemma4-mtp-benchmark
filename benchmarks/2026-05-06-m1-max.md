# Local Benchmark: MacBook Pro M1 Max 64 GB

This is a small local proof run from the repository creation session. It is not an official score.

## Environment

- Date: 2026-05-06
- Machine: MacBook Pro
- Chip: Apple M1 Max
- Memory: 64 GB
- Architecture: arm64
- OS reported by Python: macOS-26.4.1-arm64-arm-64bit
- LiteRT-LM Python package: litert-lm-api-nightly 0.11.0.dev20260422

## E2B GPU Quick Smoke

Command:

```bash
uv run gemma4-mtp-bench run \
  --model e2b \
  --backend gpu \
  --prompt-set quick \
  --rounds 1 \
  --warmups 1 \
  --output results/e2b-gpu-quick-local.json
```

Result:

| MTP | runs | mean seconds | mean chars/sec | est tokens/sec |
| --- | ---: | ---: | ---: | ---: |
| off | 1 | 2.69 | 105.3 | 55.7 |
| on | 1 | 3.43 | 88.6 | 45.6 |

Estimated MTP speed ratio: 0.82x

This quick prompt is mainly a smoke test. It is short and freeform, so it is not the best place to feel MTP.

## E4B GPU Coding

Command:

```bash
uv run gemma4-mtp-bench run \
  --model e4b \
  --backend gpu \
  --prompt-set coding \
  --rounds 1 \
  --warmups 1 \
  --output results/e4b-gpu-coding-local.json
```

Result:

| MTP | runs | mean seconds | mean chars/sec | est tokens/sec |
| --- | ---: | ---: | ---: | ---: |
| off | 2 | 76.28 | 54.4 | 17.8 |
| on | 2 | 40.79 | 101.5 | 33.2 |

Estimated MTP speed ratio: 1.87x

This run showed the expected experiential difference: the MTP run felt much faster during code generation.
